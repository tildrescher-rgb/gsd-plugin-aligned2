  <files>bin/check-gsd-release.sh</files>
  <action>
Create `bin/check-gsd-release.sh` with the following logic:

1. **Shebang and config section:**
   - `#!/bin/bash`
   - `set -euo pipefail`
   - Define constants: `REPO="gsd-build/get-shit-done"`, `VERSION_FILE="$HOME/.gsd-last-known-version"`, `GH="/opt/homebrew/bin/gh"`, `RECIPIENT="jnuyens"`, `MAIL_HOST="m1.linuxbe.com"`

2. **Network connectivity check:**
   - `curl -sf --max-time 5 https://api.github.com/zen >/dev/null 2>&1 || exit 0`
   - This hits a lightweight GitHub endpoint; if it fails (no network, DNS down, timeout), exit 0 silently

3. **Fetch latest release tag:**
   - `LATEST=$($GH api "repos/$REPO/releases/latest" --jq '.tag_name' 2>/dev/null)` — if gh fails, exit 0 silently
   - Strip leading "v" if present: `LATEST="${LATEST#v}"`
   - Validate LATEST is non-empty, exit 0 if empty

4. **First-run handling:**
   - If `$VERSION_FILE` does not exist: write `$LATEST` to it and exit 0 (no email on first run)

5. **Version comparison:**
   - Read `PREVIOUS` from `$VERSION_FILE`
   - If `$LATEST` equals `$PREVIOUS`: exit 0 (no new version)

6. **Send notification email:**
   - Compose body with:
     - New version: `$LATEST`
     - Previous version: `$PREVIOUS`
     - Release URL: `https://github.com/$REPO/releases/tag/v$LATEST`
     - Timestamp of check
   - Send via SSH to m1.linuxbe.com: `echo "$BODY" | ssh "$MAIL_HOST" "mail -s 'GSD upstream release: v$LATEST available (was v$PREVIOUS)' $RECIPIENT"`
   - This avoids local mail delivery issues by using m1.linuxbe.com's configured mail system

7. **Update version file:**
   - Write `$LATEST` to `$VERSION_FILE` (only after successful mail send)

Make the script executable: `chmod +x bin/check-gsd-release.sh`
